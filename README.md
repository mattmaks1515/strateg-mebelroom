# Стратег-советник владельца — Мебельрум (Telegram-бот)

ИИ-советник по стратегии для владельца мебельной компании «Мебельрум» (Уфа).
Живёт в Telegram, отвечает 24/7, думает на уровне внешнего консультанта:
SWOT, бизнес-модель, 5 сил Портера, разбор конкурентов, сценарии («второй салон»,
«новый город», «поднять цены», «нанять РОПа»), решения по приоритетам.

## Мозг и контекст
- `CLAUDE.md` — инструкция советника (роль, тон, принципы, рыночные якоря, фреймворки,
  формат ответа, самопроверка). Правится по блокам.
- `company.md` — контекст компании (цифры, команда, каналы, цели, конкуренты).
- `logs/decisions.md` — лог ключевых рекомендаций и решений владельца.

Бот грузит `CLAUDE.md` + `company.md` как system-prompt (см. `brain.py`).
Правишь блок → `git pull` на сервере → `systemctl restart strateg-bot`.

## Код
- `bot_worker.py` — воркер 24/7 (long-polling Telegram), whitelist по `OWNER_USER_ID`.
- `advisor.py` — вызов Claude Messages API (модель Fable 5).
- `brain.py` — сборка system-prompt из файлов + рантайм-ограничения этапа.
- `memory.py` — память диалога на пользователя (`logs/dialogs/<id>.json`).
- `telegram_io.py` — слой Telegram (вызовы, отправка с разбивкой длинных ответов).
- `common.py` — `.env`, форс IPv4, SSL (грабли VPS учтены).
- `deploy/` — systemd-сервис, `setup.sh`, инструкция по запуску на VPS.

## Зависимости
Только стандартная библиотека Python 3.9+. `pip install` не нужен.

## Этапы
- **B1 (сейчас):** живой бот в Telegram — мозг + диалог, цифры из `company.md`.
- **B2:** подтягивание агрегатов из Битрикса (выручка, чек, конверсия).
- **B3:** веб-поиск (серверный инструмент `web_search`) — разбор конкурентов в чате.
- **B4:** еженедельный дайджест рынка (блок 11).

## Секреты
Только в `.env` на сервере (шаблон — `.env.example`). В гит и в переписку не попадают.
